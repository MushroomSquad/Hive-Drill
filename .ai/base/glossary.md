# Glossary

Единый словарь терминов проекта. Агенты обязаны использовать эти термины в коде, документации и артефактах.

---

## Core domain terms

| Термин | Определение |
|--------|------------|
| **Project** | Зарегистрированный репозиторий с изолированным vault и runs. Хранится в `.ai/projects/<name>.json` |
| **Active Project** | Текущий активный проект. Задаётся через `just project switch`. Хранится в `.ai/state/current` (локально, не коммитится) |
| **Project Registry** | Список всех зарегистрированных проектов в `.ai/projects/`. Коммитится — общий для команды |
| **Project Vault** | Изолированное Obsidian-пространство проекта: `vault/projects/<name>/` |

## System terms (AI orchestration)

| Термин | Определение |
|--------|------------|
| **Blueprint** | Шаблон пайплайна: набор стадий, артефактов, правил handoff и eval-критериев |
| **Run** | Конкретный прогон blueprint'а по конкретной задаче (артефакты в `.ai/runs/<project>/<id>/`) |
| **Brief** | Структурированное описание задачи: цель, scope, ограничения, риски |
| **Plan** | Архитектурный план: варианты решения, выбранный путь, этапы, rollback |
| **Findings** | Нарратив после реализации: что сделано, долги, follow-up |
| **Skill** | Переиспользуемый micro-workflow для конкретной повторяющейся операции |
| **P0/P1/P2/P3** | Приоритет задачи, определяющий маршрутизацию агентов и моделей |
| **Cloud lane** | Маршрут через облачные модели (Claude Opus/Sonnet, Codex cloud) |
| **Local lane** | Маршрут через локальный LLM (Codex local profile → localhost:8000) |
| **Worktree** | Изолированная копия репозитория для параллельной агентной работы |
